## Codebase Patterns
- NeuronDB and other shared modules use IIFE pattern: `var X = X || (function() { ... })();` with `module.exports` for Node compatibility
- manifest.json content_scripts: only the first block (document_start) includes neuron-db.js and neuron-sync.js — neuron-site.js must be in that same block, loaded BEFORE neuron-db.js
- web_accessible_resources uses `shared/js/*` wildcard, so new files in shared/js/ are automatically accessible
- No build tools, TypeScript, or package.json — plain vanilla JS Chrome extension (Manifest V3)
- background.js uses importScripts() for shared modules
- NeuronDB cache uses semantic property names (cache.demandas, cache.config) — NOT storage key names
- Per-site storage keys are built dynamically: `neuron_${siteAlias}_demandas`, `neuron_${siteAlias}_concluidas`, `neuron_${siteAlias}_metadata`
- NeuronDB._getCurrentSiteAlias() returns the active site alias string
- NeuronSync filters per-site storage key changes using regex; only updates cache for the active site
- loading.js (document_start) is the first content script to run — NeuronDB.init() with site alias here sets context for all subsequent content scripts
- Extension pages (popup, dashboard, options, background) default to 'producao' for NeuronDB.init() — site switching comes via US-007/US-008
- NeuronDB.switchSite(alias) resets cache and reloads from storage — must be awaited before reading data
- NeuronDB.getCurrentSite() returns the active site alias (public API); _getCurrentSiteAlias() is for NeuronSync internal use
- Extension pages (popup, dashboard, options) must explicitly add neuron-site.js in their HTML script tags — it's only auto-loaded in content_scripts via manifest.json
- Site selector segmented control CSS uses `.site-btn.active[data-site="X"]` pattern — reusable across popup and dashboard
