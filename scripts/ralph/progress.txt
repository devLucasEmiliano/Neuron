## Codebase Patterns
- This is a Chrome Extension (Manifest V3) with no build tools, no package.json, no linter — quality checks are manual file reference verification
- `manifest.json` is the single source of truth for all file paths; every moved/renamed file must have its manifest reference updated
- HTML files in `modules/` use relative paths like `../../vendor/` to reach root-level directories
- `background.js` uses `importScripts()` with root-relative paths
- `chrome.runtime.getURL()` calls in JS files use root-relative paths (e.g., `modules/loading/loading.html`)
- Pre-existing line-ending changes exist in many files — only stage files you actually modified
- Use `git mv` for renames to preserve file history
- Chrome Manifest V3 `web_accessible_resources` glob `*` does NOT cross directory boundaries — `images/*` won't match `images/icons/foo.png`
- Icon files (manifest `icons`, `action.default_icon`, extension page `<img>` tags) don't need to be in `web_accessible_resources`
