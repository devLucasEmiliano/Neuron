## Codebase Patterns
- ThemeManager is a global singleton in `shared/js/theme-manager.js` - uses NeuronDB preferences store for persistence
- NeuronDB preferences: `getPreference(key)` / `setPreference(key, value)` - auto-broadcasts via NeuronSync
- NeuronSync broadcasts with `type: 'preference'` and `type: 'config'` - listeners via `onPreferenceChange(callback)` / `onConfigChange(callback)`
- Theme CSS is NOT injected into content scripts via manifest - only loaded in popup.html and options.html via `<link>` tags
- Bootstrap theming uses `data-bs-theme` attribute on `<html>` element
- Neuron custom CSS vars are defined in `shared/css/theme.css` and scoped to `[data-bs-theme]` selectors
- UI labels are in Portuguese (pt-BR)
- No npm/node build system - plain JS, validation is JSON well-formedness + shell script syntax
- Popup footer layout: `d-flex justify-content-between` with link | version | theme controls - wrap grouped controls in a nested flex div
- ThemeManager._enabled is synchronously available after init() for initial UI state
