## Codebase Patterns
- NeuronDB is an IIFE that returns a public API object, all methods are async using `init()` pattern
- DB stores use `{ keyPath: 'key' }` for key-value stores (metadata, config, preferences)
- Store records include `{ key, value, timestamp }` for key-value stores
- No build tooling (no package.json, eslint, tsconfig) - use `node -c` for syntax validation
- Script load order matters: `idb.min.js` → `neuron-db.js` → `neuron-sync.js` → consuming modules
- NeuronSync is an IIFE wrapping BroadcastChannel('neuron-sync'), exposes broadcast(), onConfigChange(), onPreferenceChange()
- NeuronDB.setConfig() and setPreference() auto-broadcast via NeuronSync when it's loaded (graceful fallback if not)
- neuron-utils.js depends on NeuronDB + NeuronSync globals — load order: idb.min.js → neuron-db.js → neuron-sync.js → neuron-utils.js
- module-factory.js depends on NeuronDB + NeuronSync globals — load order: idb.min.js → neuron-db.js → neuron-sync.js → module-factory.js → consuming module
- Content scripts are injected via manifest.json `content_scripts` entries
- Extension pages (options, popup) load scripts via `<script>` tags in HTML
- Service worker (background.js) uses `importScripts()` for dependencies
- The full user config is stored under key `neuronUserConfig` — `NeuronDB.getConfig('neuronUserConfig')` returns the full object directly (not wrapped in `{ key: value }`)
- Theme preference is stored under key `theme` in preferences store — `NeuronDB.getPreference('theme')` / `NeuronDB.setPreference('theme', value)`
- Anti-FOUC IIFE can't use IndexedDB (async), so it falls back to `ThemeManager.getSystemTheme()` — corrected once `init()` runs
- notificacoes.js stores filtro usuario preference under key `filtroUsuarioAtivado` in NeuronDB preferences store
- notificacoes.js stores dashboard refresh signal under key `dashboardRefreshSignal` in NeuronDB preferences store

- Exposes broadcast(changeType, key, newValue) to send change notifications
