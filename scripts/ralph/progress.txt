## Codebase Patterns
- NeuronDB is an IIFE that returns a public API object, all methods are async using `init()` pattern
- DB stores use `{ keyPath: 'key' }` for key-value stores (metadata, config, preferences)
- Store records include `{ key, value, timestamp }` for key-value stores
- No build tooling (no package.json, eslint, tsconfig) - use `node -c` for syntax validation
- Script load order matters: `idb.min.js` → `neuron-db.js` → `neuron-sync.js` → consuming modules
- NeuronSync is an IIFE wrapping BroadcastChannel('neuron-sync'), exposes broadcast(), onConfigChange(), onPreferenceChange()
- NeuronDB.setConfig() and setPreference() auto-broadcast via NeuronSync when it's loaded (graceful fallback if not)
- Content scripts are injected via manifest.json `content_scripts` entries
- Extension pages (options, popup) load scripts via `<script>` tags in HTML
- Service worker (background.js) uses `importScripts()` for dependencies

- Files changed: shared/js/neuron-sync.js (new), shared/js/neuron-db.js
- **Learnings for future iterations:**
  - BroadcastChannel messages are received by all other contexts but NOT the sender — each context only hears changes from other contexts
  - The `typeof NeuronSync !== 'undefined'` guard in NeuronDB means neuron-sync.js is optional — it can be loaded after neuron-db.js without breaking anything
  - When replacing chrome.storage.onChanged listeners in future stories, use NeuronSync.onConfigChange() or NeuronSync.onPreferenceChange() and ensure neuron-sync.js is loaded in that context
