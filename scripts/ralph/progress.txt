## Codebase Patterns
- Theme CSS custom properties (`--neuron-*`) are defined in `shared/css/theme.css`; page-specific CSS files should use fallback values like `var(--neuron-primary, var(--bs-primary, #0d6efd))` so they degrade gracefully when theme is disabled
- ThemeManager in `shared/js/theme-manager.js` controls theme enable/disable for all extension pages (popup, options, content scripts)
- `_disableTheme()` removes `data-bs-theme` attr and disables `theme.css` link; `applyTheme()` re-enables both
- Use `clearTimeout`/`setTimeout` pattern in ThemeManager to delay stylesheet disable for smooth CSS transitions
- UI for theme toggle state is managed by `updateThemeEnabledUI()` in both popup.js and options.js
- No build system / linter / typecheck tooling - quality check is `node --check` for JS syntax
