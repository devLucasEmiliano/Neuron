## Codebase Patterns
- This is a Chrome Extension (manifest v3) — no package.json, no npm scripts, no traditional build/lint/typecheck
- Shared CSS/JS goes in `shared/css/` and `shared/js/`, loaded via `manifest.json` content_scripts
- CSS variables are defined in `:root` in `tratar-novo.css` and `sic-tratar-style.css` (e.g., `--cor-card`, `--raio-borda`)
- Neuron theme variables (e.g., `--neuron-bg-primary`) may also be available at runtime
- Module-specific CSS should NOT duplicate rules that exist in shared CSS files
- Selectize dropdowns use classes: `.selectize-dropdown`, `.selectize-control`, `.selectize-input`
- ASP.NET UpdatePanel can recreate DOM elements — use MutationObserver for resilience
- Quality check = valid JSON (manifest.json) + valid JS syntax (no build step)
