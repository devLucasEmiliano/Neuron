## Codebase Patterns
- This is a Chrome Extension with plain JS (no TypeScript, no build system, no package.json)
- Theme management is handled by ThemeManager class in shared/js/theme-manager.js
- Options page uses sidebar navigation pattern with lazy section initialization (setupSidebarNavigation)
- The options page loads dashboard.js and dashboard.css alongside options.js
- ThemeManager.cycle() cycles through light → dark → system themes
- NeuronDB and NeuronSync are the data layer; config is stored via chrome.storage
- CSS uses Bootstrap 5 with custom Neuron theme classes (text-neuron-primary, btn-neuron-primary, etc.)
- Dedicated dashboard page at modules/dashboard/ is separate from options embedded dashboard at modules/options/dashboard.js
- New extension pages should load scripts in order: bootstrap.bundle → chart.min → neuron-db → neuron-sync → neuron-utils → theme-manager → page.js
- NeuronSync only handles config/preference changes; for demandas/concluidas data changes, use chrome.storage.onChanged directly
- refreshDashboard() in dashboard.js is the central refresh function — extend it for charts, tables, filters
- getThemeColors() in dashboard.js returns all chart colors; extend it when adding new chart types
- Chart instances (chartStatus, chartPrazos, etc.) are module-scoped; on theme change, destroy+recreate them
- computeLocalStats(demandas, concluidasSet) mirrors NeuronDB.getStats() for filtered subsets — use it when filtering is needed
- Filter state: currentFilter ('all'|'mine') and currentUser are module-scoped in dashboard.js; persisted via NeuronDB preferences
- Use NeuronUtils.escapeHtml (window.NeuronUtils.escapeHtml) for XSS-safe HTML rendering — already loaded on all extension pages
- renderDemandasTable(stats) renders the full table from stats.demandas — extend for pagination
- Search/sort state (searchQuery, sortColumn, sortDirection) is module-scoped; renderDemandasTable(null) re-renders with lastStats
- searchDemandas() and sortDemandas() are composable pipeline functions: search filters first, then sort applies
- Pagination state (currentPage, itemsPerPage) is module-scoped; renderPagination() renders Bootstrap pagination controls
- Pagination resets to page 1 on search, filter, or items-per-page change; itemsPerPage persists via NeuronDB preference 'dashboardItemsPerPage'
- Demanda objects have an `href` field with the full Fala.BR URL (may be null); use it for linking to the demand page
- For row action buttons, use event delegation on the table since tbody content is re-rendered frequently
